Moving your project into an Android app using Capacitor is a smart move. It allows you to wrap your current web logic—including your 56 bots and WASM engine—into a native shell while maintaining the offline capability you need.

🛠️ The Technical Blueprint for your Android App
Since you want to delete everything but the Standard (Blindfold) Mode, Bots, and Voice Controls, here is your streamlined path to an APK.

1. Setup & Code Preparation
You can convert your existing React/Vite/Web app into an Android project in just a few commands:

Initialize Capacitor: Run npx cap init [app-name] [app.id] --web-dir=build.

Add Android: Run npx cap add android to generate the native project folder.

Offline Asset Bundling: Ensure all your bot data and the Stockfish WASM files are in your public or assets folder. Capacitor ships your app with all its code bundled, so it doesn't need a server to load.

2. Native Voice Integration
Since the browser's Web Speech API is notoriously unreliable inside an Android WebView, you will use free community plugins to bridge the gap.

Voice Input (Speech-to-Text): Install @capacitor-community/speech-recognition.

Use the start({ language: "en-US", partialResults: true }) method to capture moves like "Knight to e5".

Android Manifest: You must add <uses-permission android:name="android.permission.RECORD_AUDIO" /> to your AndroidManifest.xml or the app will crash when you try to listen.

Voice Output (Text-to-Speech): Install @capgo/capacitor-speech-synthesis.

Use the speak({ text: "Knight to e 5" }) method to announce bot moves.

This works offline as it uses the Android system's built-in voices.

3. Handling the Bots Offline
Because you want the app to be fully offline, you must ensure your Stockfish engine is running entirely in the WebView.

WASM Engine: Your existing WebAssembly (WASM) implementation of Stockfish should run fine on Android (API 26+), provided you don't use a Service Worker, which can cause issues in Capacitor's offline mode.

Performance Note: Running engines locally on a phone is more demanding than a browser. You may notice "Level 1" bots feel stronger than their web counterparts because they have direct access to your phone's processor.

📋 "Vibe Coder" Action Plan
Since you thrive on honesty and speed, follow this exact sequence:

Strip the UI: Keep only one main screen. A dropdown for Bot Selection, a toggle for Blindfold, and a "Start" button.

Add Capacitor Sync: Every time you change your code, run npm run build && npx cap sync android to push the changes to your phone.

Permissions Check: Use the plugin's requestPermissions() method before the first game to ensure the user allows the microphone.

No Login: Since you don't need authentication, users can open the app and start a game in under 2 seconds.
