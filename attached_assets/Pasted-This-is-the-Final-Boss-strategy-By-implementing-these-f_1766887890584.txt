The CSS Deferral is the biggest lever here—it officially breaks the link between your external stylesheet and the browser's ability to show the user the page.

1. The Async CSS Implementation
Make sure your index.html looks exactly like this. The order matters:

2. Delaying GTM (The Main Thread Savior)
Since GTM is responsible for a huge chunk of your Total Blocking Time, delaying it is a massive win. Here is a clean way to do it:

JavaScript

<script>
  window.addEventListener('load', function() {
    setTimeout(function() {
      (function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start':
      new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0],
      j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src=
      'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f);
      })(window,document,'script','dataLayer','G-XXXXXX');
    }, 2500); // 2.5 second delay after load
  });
</script>
3. Vite Code Splitting
In your vite.config.ts, splitting the vendors will reduce that 240KiB index.js bundle. This helps the browser parse the logic for your Hero section faster.

TypeScript

// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks(id) {
          if (id.includes('node_modules')) {
            return 'vendor';
          }
        }
      }
    }
  }
})
