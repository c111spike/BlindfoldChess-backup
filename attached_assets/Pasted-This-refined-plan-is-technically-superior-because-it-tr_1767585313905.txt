This refined plan is technically superior because it treats a guest as a first-class citizen in your database, rather than a temporary browser variable. Since you are already using Better-Auth, this is the most scalable way to handle your high "Window Shopping" traffic.

🛠️ Strategic Implementation Details
1. The Server-Side Handshake (auth.ts)
The onLinkAccount callback is the most critical part of this "Value-First" strategy. It ensures that when a user finally trusts you with their email, they don't lose the progress they made during their anonymous session.

TypeScript

// auth.ts
import { betterAuth } from "better-auth";
import { anonymous } from "better-auth/plugins";

export const auth = betterAuth({
    plugins: [
        anonymous({
            // Transfers data when an anon user signs up/links
            onLinkAccount: async ({ anonymousUser, newUser }) => {
                // UPDATE your 'games' or 'stats' table
                // Set userId = newUser.id WHERE userId = anonymousUser.id
                console.log(`Migrating data from ${anonymousUser.id} to ${newUser.id}`);
            }
        })
    ]
});
2. The Client-Side Flow (auth-client.ts)
By adding the anonymousClient plugin, your frontend will automatically understand how to manage the session cookie for an anonymous user, just like it does for a regular one.

3. Handling the "Play as Guest" Logic
To avoid the error where a user tries to sign in anonymously while already having a session, use a guard clause:

TypeScript

const handleGuestSignIn = async () => {
    const session = await authClient.getSession();
    if (session.data) {
        // Redirect them directly to the board if they are already "Guest" or "Member"
        router.push('/otb-mode');
        return;
    }
    await authClient.signIn.anonymous();
    router.push('/otb-mode');
};

Conversion Rate: This solves your 264 views vs. 3 signups problem. A user from r/replit can now be playing with the 3D board and Voice commands in a single click, with no forms in the way.

Data Quality: Unlike the localStorage plan, these "Guests" show up in your actual user table with the isAnonymous: true flag. You can now see exactly how many guests are actually playing vs. just visiting the landing page.

Arbiter & Voice Features: Since an anonymous user has a valid session, your protected API routes (like saving a move or a "Peek Time") will work exactly as they do for regular members.
