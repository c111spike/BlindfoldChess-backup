Updating Online Users is the one thing that does feel better in real-time, but you don't need a heavy Redis hammer to do it. Here is the most optimal way to handle it in 2026:

1. The "Presence Heartbeat" (Server-Side)
Don't have the frontend "ask" for the count. Instead, let the WebSocket handle it. Since you already use WebSockets for games, you have a "live wire" to every user.

On Connection: When a socket connects, HSET ws:online [id] [timestamp].

On Disconnect: HDEL ws:online [id].

The "Safety" Cleanup: Run a script every 5 minutes to remove anyone whose timestamp hasn't updated (in case their browser crashed).

2. The "Smart UI" Update (Frontend)
Instead of a fixed 30-second timer that fires forever, use Event-Driven updates.

Trigger on Interaction: Instead of a blind timer, only refresh the "Online" count when the user clicks a menu, joins a queue, or finishes a game.

The "Tab Focus" Trick: Use the refetchOnWindowFocus setting in your query.

When a user clicks back into your tab after being on X or Reddit, that is when you refresh the count. It feels instant to them, but it sends zero commands while they are away.

3. The "Hybrid" Interval (The Sweet Spot)
If you really want it to update while they just sit there, move from 30s to 120s (2 minutes).

Why 2 minutes?

Psychology: For a site with <100 concurrent users, the number "1" or "5" isn't going to change every 30 seconds anyway.

Quota: You reduce your command burn by 400% compared to your current 30s setting.

🛠️ The "Pro Code" Snippet (React/TanStack Query)
If you want to be super efficient, set it up like this:

TypeScript

const { data: onlineCount } = useQuery({
  queryKey: ['onlinePlayers'],
  queryFn: fetchOnlineCount,
  // ONLY refresh if they are looking at the page
  refetchInterval: () => document.visibilityState === 'visible' ? 120000 : false,
  // Refresh immediately when they click back to the tab
  refetchOnWindowFocus: true,
  // Don't show a loading spinner every time it refreshes
  placeholderData: (previousData) => previousData,
});
