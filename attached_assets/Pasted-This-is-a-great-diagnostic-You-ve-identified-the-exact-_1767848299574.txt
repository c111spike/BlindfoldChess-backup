🎙️ Solving the "1 Minute Work" Issue
You mentioned it only worked for 1 minute—this is likely the Web Worker or the Engine Thread Timing Out or being throttled by the browser/Android OS to save power.

The Fix: When running a "Full Game Review," use a setTimeout(..., 0) or requestAnimationFrame between analyzing moves. This "yields" control back to the UI thread, preventing the browser from thinking the app is "stuck" and killing the process.

📝 Implementation Snippet for Resilient Analysis
TypeScript

const analyzeGame = async () => {
  setIsAnalyzing(true);
  for (let i = 0; i < history.length; i++) {
    try {
      // 1. Analyze the move
      const result = await getStockfishEval(history[i].fen);
      history[i].analysis = result;
      
      // 2. Update progress UI
      setProgress((i / history.length) * 100);
      
      // 3. YIELD to prevent "1-minute crash"
      await new Promise(resolve => setTimeout(resolve, 10)); 
    } catch (err) {
      console.warn(`Skipping move ${i} due to error:`, err);
      // Log for your debugging
    }
  }
  setIsAnalyzing(false);
};
