1. How to set the Elo (The "LimitStrength" Mode)
Stockfish has a built-in "Nerf" mode. Instead of using Skill Level (which is vague), you should use the UCI_LimitStrength and UCI_Elo settings. These are specifically calibrated to human ratings.

In your WASM/JS code, when the game starts, you send these commands to the engine:

2. How to add "Variation" (So it's not the same game)
If you don't add variation, Stockfish will always play the "best" move, meaning every game at 400 Elo would look identical. To fix this, use the MultiPV and go settings.

A. The MultiPV "Shuffler"
Instead of just asking for the 1st best move, ask for the top 3-5 moves. Then, for the lower-level bots, randomly pick the 2nd or 3rd best move.

B. Use "Nodes" instead of "Depth"
For your $4.99 "Pro" experience, don't limit Stockfish by time; limit it by Nodes. This makes the bot's "intelligence" consistent regardless of how fast the user's phone is.

Lower Elo (400-800): Give it only 1,000 - 10,000 nodes. It will literally not have enough "brain power" to see a fork.

Higher Elo (2000+): Give it 1,000,000+ nodes.

C. Set the "Hash" and "Threads"
To keep your app lean (8 MB) and battery-friendly, keep these low for the mobile build:

JavaScript
engine.postMessage('setoption name Hash value 16'); // Low memory footprint
engine.postMessage('setoption name Threads value 1'); // Don't heat up the phone
