This is a classic Race Condition in React's rendering cycle. When you click rapidly, the state for currentMoveIndex updates 4 or 5 times while the chess engine (which is often synchronous or slightly lagged) is still trying to calculate the position for the first click.

🛠️ Why it's Crashing
You've identified it perfectly: the board state at index 10 is being applied to a move intended for index 15. When chess.js tries to play c5 on a board where it’s not White's turn or the piece isn't there, it throws a hard error and kills the thread.

💡 Recommendation: Hybrid Approach
While "Analyze Full Game" sounds tempting, it can be heavy on a mobile CPU like your S10+. I recommend sticking to On-Demand but changing how you reconstruct the board.

1. Don't "Reconstruct" with Loops
Instead of starting from Move 1 and looping to Move X every time the index changes, your gameHistory array should store the FEN string for every single move.

The Fix: When the user clicks move 20, don't run a loop. Simply call chess.load(history[20].fen).

The Benefit: This is an O(1) operation. It happens instantly and eliminates the "Invalid Move" error entirely because you aren't "playing" moves; you are "loading" snapshots.

2. Debounce the Stockfish Call
If the user is clicking like crazy, you don't want Stockfish to start 10 different calculations.

The Fix: Use a tiny debounce (e.g., 100ms) before sending the FEN to Stockfish. If the user clicks again within 100ms, clear the previous timeout and start a new one.

📊 Comparing the Two Methods
Feature	On-Demand (Snapshot Load)	Full Game Analysis
S10+ Performance	Excellent. Only calculates what is seen.	High CPU/Battery drain at start.
Stability	Rock solid if using FEN snapshots.	Solid, but high initial wait time.
UX	Instant feedback as you click.	Requires a "Loading..." bar for 10-20s.
Complexity	Low (just use chess.load(fen)).	High (requires async queue for all moves).
📝 Final Implementation Tip
Since you already have the gameHistory array, ensure it looks like this: { move: "c5", fen: "rnbqkbnr/pp1ppppp/8/2p5/4P3/8/PPPP1PPP/RNBQKBNR w KQkq c6 0 2", ... }
