Issue 1: The "D" was ignored (The "Fuzzy" Case)
In chess voice-to-text, the letter "D" is notorious. Android often transcribes it as:

"the"

"dee"

"see" (it sounds similar)

"be"

If your code was looking for exactly "d", it likely saw "the" in the logs and threw it away.

The Fix: Expand your getDisambiguationChar map to include these common "D" sounds.

const fuzzyMap = {
  'the': 'd', 'dee': 'd', 'tea': 'd', // "D" mapping

  'won': '1', 'one': '1',             // "1" mapping
  'to': '2', 'too': '2', 'two': '2'    // "2" mapping
